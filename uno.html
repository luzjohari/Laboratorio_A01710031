En aprendizaje automÃ¡tico y ciencia de datos, NaÃ¯ve Bayes es un algoritmo simple pero ampliamente utilizado. Este es un algoritmo clasificador basado en probabilidad. La base de este modelo es el teorema de Bayes:

$P(C \mid x) = \frac{P(x \mid C) \cdot P(C) }{ P(x) }$

Donde,

  • $P(C \mid x)$: Probabilidad posterior de ocurrencia del evento (clase) $C$, dada la observaciÃ³n $x$
  • $P(x \mid C)$: Probabilidad de observar $x$ si $C$ es verdad
  • $P(C)$: Probabilidad de observar $C$
  • $P(x)$: Probabilidad de observar $x$

En pocas palabras, el clasificador NaÃ¯ve Bayes clasifica una instancia calculando el posterior de cada clase, dada la instancia; $P(C \mid x)$, asignando la predicciÃ³n a la clase con el mayor posterior. En la prÃ¡ctica, la probabilidad posterior puede ser complicada de calcular, por lo tanto se hacen algunas (arriegadas) suposiciones, de ahÃ­ el nombre: NaÃ¯ve Bayes.

Para adquirir mayor conocimiento de como funciona, puedes ver los siguientes vÃ­deos:

  • Multinomial Naive Bayes para probabilidades discretas
  • Gaussian Naive Bayes para probabilidades continuas

Entrenamiento del modeloÂ¶

Entrenar un modelo de NaÃ¯ve Bayes es muy simple. Pero antes de llegar allÃ­, debemos establecer la distribuciÃ³n en la que basamos las probabilidades. Recordemos el Teorema de Bayes:

$$P(C \mid x) = \frac{P(x \mid C) \cdot P(C) }{ P(x) }$$

Donde,

  • $P(C \mid x)$: Probabilidad posterior de ocurrencia del evento (clase) $C$, dada la observaciÃ³n $x$
  • $P(x \mid C)$: Probabilidad de observar $x$ si $C$ es verdad
  • $P(C)$: Probabilidad de observar $C$
  • $P(x)$: Probabilidad de observar $x$

Para encontrar las probabilidades a posteriori $P(C \mid x)$, necesitamos calcular la probabilidad $P(x \mid C)$ y el a priori $P(C)$.

Podemos obtener esta probabilidad si conocemos la distribuciÃ³n de $x$. Comunmente se asume una distribuciÃ³n gaussiana (normal) para todas las clases, ya que es una buena aproximaciÃ³n para muchas otras distribuciones.

La distribuciÃ³n normal requiere dos parÃ¡metros: $\mu$ y $\sigma^2$. Dado que no conocemos los valores reales de estos parÃ¡metros debemos estimarlos a partir de las muestras actuales.

InÂ [Â ]:
import matplotlib.pyplot as plt

def normalpdf(n, mu=0, sigma=1):
    # Funcion para estimar PDF (Probability Density Function) dado que hay una distribucion Normal(mu, var) evaluado con x
    sd = np.sqrt(sigma)
    pdf = (np.e ** (-0.5 * ((n - mu)/sd) ** 2)) / (sd * np.sqrt(2 * np.pi))
    return pdf


# Define the parameters of the Gaussian function
mu = 1  # mean
sigma = 3  # standard deviation

xs = np.linspace(-5, 5, 100)
ys = normalpdf(xs, mu, sigma)

# Generate plot
plt.title('Normal PDF')
plt.plot(xs, ys, label = 'mu='+str(mu)+', sigma='+str(sigma))
plt.legend() # incluir label
plt.grid(True)
plt.show()

ClasificaciÃ³nÂ¶

El proceso de clasificaciÃ³n en un clasificador (Gaussian) NaÃ¯ve Bayes se realiza calculando el posterior para todas las clases dados los datos actuales.

Nota: $\propto$ es aquÃ­ un sÃ­mbolo que se lee como "proporcional a".

Ajuste logarÃ­tmico por UnderflowÂ¶

Volviendo al punto de clasificaciÃ³n, para calcular la probabilidad posterior $P(C_i\mid x)$ se suele sustituir la expresiÃ³n original:

$$P(C_i\mid x) \propto P(C_i)\prod_{k=1}^{K}P(x_k \mid C_i)$$

por la siguiente expresiÃ³n:

$$\log P(C_i\mid x) \propto \log P(C_i) + \sum_{k=1}^{K} \log P(x_k \mid C_i)$$

La razÃ³n de este cambio es realizar un ajuste para tomar en cuenta un problema potencial que puede suceder cuando se trabaja con cantidades muy "pequeÃ±as" (e.g., probabilidades). En pocas palabras, al multiplicar cantidades pequeÃ±as se puede tener como resultado valores demasiado pequeÃ±os para ser almacenados apropiadamente en memoria (leer sobre Underflow).

Para mitigar este problema, el ajuste por desbordamiento toma el logaritmo de las probabilidades y realiza una suma en lugar de una multiplicaciÃ³n. Al trabajar con logaritmos, probabilidades "pequeÃ±as" se transforman en valores mÃ¡s manejables, lo que reduce el riesgo de desbordamiento.

  • Nota 1: Recuerda que $\log(ab)= \log a + \log b$
  • Nota 2: TambiÃ©n recuerda que $\exp(\log x) = x$

El proceso de clasificaciÃ³n de este modelo se realiza simplemente calculando la probabilidad log-posterior y observando quÃ© clase obtiene el mayor valor.

El siguiente cÃ³digo muestra el proceso de clasificaciÃ³n:

def predict(X):
    predictions = []

    for i in X.index: # Loop through each instances

        ClassLikelihood = []
        instance = X.loc[i]

        for cls in clases: # Loop through each class

            FeatureLikelihoods = []
            FeatureLikelihoods.append(np.log(prior[cls])) # Append log prior of class 'cls'

            for col in x_train.columns: # Loop through each feature

                data = instance[col]

                mean = means[col].loc[cls] # Find the mean of column 'col' that are in class 'cls'
                variance = var[col].loc[cls] # Find the variance of column 'col' that are in class 'cls'

                Likelihood = normalpdf(data, mean, variance)

                if Likelihood != 0:
                    Likelihood = np.log(Likelihood) # Find the log-likelihood evaluated at x
                else:
                    Likelihood = 1/len(train)

                FeatureLikelihoods.append(Likelihood)

            TotalLikelihood = sum(FeatureLikelihoods) # Calculate posterior
            ClassLikelihood.append(TotalLikelihood)

        MaxIndex = ClassLikelihood.index(max(ClassLikelihood)) # Find largest posterior position
        prediction = clases[MaxIndex]
        predictions.append(prediction)

    return predictions

En esta funciÃ³n predict(X), el log-prior ($\log P(C_i)$) se calcula en la lÃ­nea 12 y las probabilidades logarÃ­tmicas $\log P(x_k\mid C_i)$ (independientes) se calculan en las lÃ­neas 21 y 24.

La probabilidad log-posterior de la clase dados los datos $\log P(C_i \mid x)$ se calcula sumando $\log P(C_i)$ y $\log P(x_k\mid C_i)$ en la lÃ­nea 30.

Finalmente, se asigna la predicciÃ³n de clase en las lÃ­neas 33 y 34 buscando el mÃ¡ximo valor log-posterior.
